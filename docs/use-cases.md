# Use Cases

BackupSDK can be integrated into a wide variety of Android applications that need to preserve user data or synchronize information across devices.

Below are some common scenarios where BackupSDK can simplify development.

---

## Games

Games often need to preserve player progress between sessions and devices.

Example data:

- Player level
- Coins
- Inventory
- Achievements
- Game settings

---

## Shopping Applications

Shopping applications can preserve user preferences and shopping information.

Example data:

- Favorite products
- Shopping cart
- Wishlist
- Delivery preferences
- Saved addresses

---

## Fitness Applications

Fitness applications continuously generate user data that should not be lost.

Example data:

- Workout history
- Daily goals
- Calories burned
- Personal records
- User preferences

---

## Travel Applications

Travel applications often store personalized information that users expect to access across multiple devices.

Example data:

- Favorite destinations
- Saved routes
- Travel plans
- Offline locations
- Personal preferences

---

## Educational Applications

Educational applications can synchronize learning progress between devices.

Example data:

- Completed lessons
- Quiz scores
- Study progress
- Saved notes
- Learning preferences

---

## Social Applications

Social applications can preserve user-specific settings and personalized content.

Example data:

- Profile settings
- Notification preferences
- Saved drafts
- Favorite content
- User customization

---

## Why BackupSDK?

Instead of building and maintaining a custom cloud backup infrastructure, developers can integrate BackupSDK and immediately benefit from:

- Secure cloud backup and restore
- User-based data synchronization
- Lightweight Kotlin SDK
- Node.js backend server
- MongoDB storage
- Developer Portal
- REST API
- Minimal integration effort

BackupSDK reduces development time while providing a complete, ready-to-use backup solution.
